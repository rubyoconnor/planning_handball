Describing Relevant Implications

Aesthetics

Products are expected to look good. This is because it makes the product more enjoyable and easier to use. A basic method to get an aesthetic result is CRAP. Contrast (big/small, dark/light, colourful/plain). Repetition (using the same visual elements). Alignment (precise formatting). Proximity (same visual elements together) (grouping). If you follow CRAP the website will most likely look visually appealing.

Accessibility

The implication of web accessibility is making sure that a website is accessible to all people (as much as possible). Many people have vision problems or mobility problems or people may be accessing them with very limited bandwidth. Search engines also need to be able to find the website and categorise it correctly. To make the website usable to all users these are some things that should be included. All images need descriptive “alt” tags so a screen reader can read them out or a search engine can know what they are about. Use in body CSS font-size 100% and use “em” measures for fonts and finally make sure colours are okay for colour blind users.

Future Proofing

Future Proofing is keeping the website up to date. This means that content needs to be easily changed so that old content can be removed and new content can be added. In many cases, the user of the site wouldn’t have the knowledge to be able to maintain the website from the code. This means that it is essential for the users to be able to maintain the website from the interface. This user needs forms and option buttons that are only available to them. Through making a website that uses a database to store content and making the site available to manage the content, the website would be able to be kept up to date.

Functionality

The functionality implication means that the website needs to work properly. It should be consistent across different browsers and mobile devices. It also needs to load all of its assets quickly. To achieve this images need to be compressed, the site should be tested for different screens and finally moved off your device to another and tested.
